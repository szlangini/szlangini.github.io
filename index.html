Erdős Number GraphDB Implementation

Overview:

Within the framework of a database project at the university, this project was implemented by our team of three students. The goal was to implement the first three layers of Erdős numbers using a graph database. We had to crawl the data with a self-written crawler. Afterwards we imported the data into Neo4j, to be able to do analyses, for example we investigated clusters in authorship. Finally, we built a GUI that allows external users to enter an author and then get a graph of the author relationship with Erdős.

A module description for DBPro can be found here

Utilized:

The second image describes the architecture of this project. We used a java-based crawler to gather the data. Additionally it was exported as CSVs to Neo4j. The Data was then queried in both the Neo4j browser (Cypher) and our self-developed webUI. The webUI was written in HTML, CSS and javaScript. However, we mainly used Alchemy.js as a framework for displaying graphs.

Scoring Model for DSL Syntax Choice

Overview:

During my bachelor's thesis I dealt with domain-specific languages (DSL). Mainly it was about differing syntaxes. Different syntaxes have different advantages related to a use case. I developed a scoring model which presents a catalogue of 20 criteria, taking into account the technical implementation, the communication basis and the economic orientation of the user. The goal is to identify a recommended syntax (graphical and/or textual) for the individual problem. I have digitally implemented this model in order to make it accessible for outsiders and to provide decision support.

The methodology can be tested out here

Utilized:

The methodology itself was developed by me. However it was implemented using HTML, CSS (Bootstrap) and javaScript. In order to understand DSLs I have tested and implemented cases within XText and Visual Studio

Beesis: Eventmanagement WebApp

Overview:
